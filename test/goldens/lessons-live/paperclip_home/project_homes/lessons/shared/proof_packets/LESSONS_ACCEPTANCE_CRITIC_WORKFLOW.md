# Lessons Acceptance Critic Workflow

This file owns the critic turn shape.

Normal Lessons lane order and same-issue handoff law live in
`paperclip_home/project_homes/lessons/shared/AUTHORITATIVE_LESSONS_WORKFLOW.md`.

Use `lessons_content_standards/LESSONS_ACCEPTANCE_CRITIC_CRITERIA.md` for:

- what must be true before full review
- what `accept` and `changes requested` mean
- who should own remediation

## Read In This Order

1. the active issue and latest packet comment
2. the current packet files
3. `lessons_content_standards/LESSONS_ACCEPTANCE_CRITIC_CRITERIA.md`
4. `proof_packets/LESSONS_PUBLISH_AND_FOLLOWTHROUGH_WORKFLOW.md` when publish
   or followthrough claims are in scope
5. `proof_packets/LESSONS_EXISTING_LESSON_MAINTENANCE_WORKFLOW.md` when the
   issue is bounded maintenance work

## Next Owner

Use the shared same-issue handoff rule in
`AUTHORITATIVE_LESSONS_WORKFLOW.md`.

- On `accept`, the next owner is the next normal lane named there.
- On `changes requested`, the next owner is the earliest owner who can fix the
  problem.
- If the correct owner is unclear, the next owner is `Lessons Project Lead`.
- If ownership is not changing yet, leave the verdict and keep the issue where
  it is.
- Do not take over normal packet work in the critic lane.
- If the only thing left is stale-file cleanup in the touched area, and the
  current packet already makes the cleanup obvious, clear those files in the
  same pass.
- After that cleanup, leave one short comment saying what you cleared and then
  leave `accept` or `changes requested`.
